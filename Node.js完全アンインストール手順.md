Node.jsを完全にアンインストールする場合、単に「プログラムの追加と削除」から消すだけでは、設定ファイルやキャッシュが残り、**再インストール時に不具合を引き起こす原因**になります。

以下に、クリーンな状態に戻すための完全アンインストール手順をまとめました。

# ---

**Node.js 完全アンインストール手順 (Windows)**

### **1\. プログラムのアンインストール**

まず、標準的なアンインストール作業を行います。

1. **設定** \> **アプリ** \> **インストールされているアプリ**（またはコントロールパネルの「プログラムのアンインストール」）を開きます。  
2. **Node.js** を探し、「アンインストール」を実行します。

### **2\. 残存ディレクトリの物理削除**

インストーラーでは削除しきれない設定ファイルやキャッシュを手動で削除します。エクスプローラーのアドレスバーに以下のパスを貼り付けて移動し、フォルダごと削除してください。

| 削除対象のパス | 内容 |
| :---- | :---- |
| C:\\Program Files\\nodejs | 本体インストール先（残っている場合） |
| C:\\Program Files (x86)\\nodejs | 32bit版を入れていた場合 |
| %AppData%\\npm | npmのグローバル設定とバイナリ |
| %AppData%\\npm-cache | npmのキャッシュデータ |
| %UserProfile%\\.npmrc | npmの設定ファイル（ファイル単体） |
| %LocalAppData%\\temp\\npm-\* | npmの一時ファイル |

**Tip:** %AppData% などの変数は、エクスプローラーのアドレスバーにそのまま打ち込むと該当フォルダにジャンプできます。

### **3\. 環境変数の確認**

Node.jsへのパスが残っていると、コマンド実行時にエラーが発生することがあります。

1. 「Windowsキー」を押し、**「環境変数を編集」**（システム環境変数の編集）と入力して開きます。  
2. \*\*「環境変数」\*\*ボタンをクリックします。  
3. 「ユーザー環境変数」および「システム環境変数」の両方から **Path** を選択し、「編集」をクリックします。  
4. 以下の項目が残っていれば削除します。  
   * C:\\Program Files\\nodejs\\  
   * C:\\Users\\\<ユーザー名\>\\AppData\\Roaming\\npm

### **4\. 再起動**

念のため、Windowsを再起動して設定を完全に反映させます。

### **5\. 最終確認**

コマンドプロンプトやPowerShellを開き、以下のコマンドを入力してエラー（「認識されていません」）が出ることを確認してください。

Bash

node \-v  
npm \-v

## ---

**補足：chocolatey や nvm-windows を使っている場合**

もし nvm-windows（バージョン管理ツール）を利用してインストールした場合は、上記の手順ではなく以下のコマンドを使用してください。

* **nvm-windowsの場合:** nvm uninstall \<version\>  
* **Chocolateyの場合:** choco uninstall nodejs

これでサーバーは完全にクリーンな状態になります。
